Choose an option how the build number should be provided to Silk Central:

  • Don't care about a build number:  Depending on how the execution plan is configured, the specified build number of Silk Central is used. This option should also be used if your execution plans are configured to 'Read from Build Information File'. For detailed information, see the Silk Central Help.
  • Use last build number added on Silk Central: Use the latest build number for this product and version configured in Silk Central. For more information on how to configure products and versions, see Silk Central Help.
  • Use build number from this job: Use the build number from the current Jenkins job and create it in Silk Central if it doesn't exist.
  • Use build number from upstream job: Use the build number of the selected Jenkins upstream job and create it in Silk Central if it doesn't exist.
