Specify the parameters needed by the execution definition ids, when they are running on your SilkCentral Test Manager system.
Insert them in the "name=value" format. Shell-like environment variable expansions work in this field for the parameters values, by using the ${VARIABLE} syntax. You can enter more than one parameter by using a newline separated list of parameters. E.g.:
param1=value
param2=another value
param3=${VARIABLE}

Note: The Parameter name is case sensitive and the parameters are set on all execution definitions specified on plugin configuration.
